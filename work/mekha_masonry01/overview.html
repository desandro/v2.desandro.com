Overview

This package has the Masonry 0.8 script (both minified and unminified) and several demos to show you what it's capable of.

In response to your images

  1. The Masonry 0.8 script can arrange elements with different widths.
  2. Each item needs to be sized via CSS. Masonry will only handle the layout of the items, not sizing them differently. Calling this version of Masonry is similar to the previous, on there are a couple options:
    1. singleMode: false, // false by default, set this to true if you want to use the old version of Masonry where ever item div is the same width and save a couple milliseconds of rendering time.
    2. columnWidth: 200, // size in pixels of each column in the grid. If not size is provided, it defaults to the size of the first element in the wrapper div
    3. itemSelector: undefined, // use this if you want to specify which class should be targeted when being arranged
    4. resizeable: true, // true by default. This option will resize the items when browser is resized
    5. newContent: undefined, // this is an advanced feature, useful when appending content. Pass in an element. if newContent is set, then itemSelector is also required.
    6. saveOptions: true // another advanced feature, by default Masonry saves your options, This is to be used when using newContent and you do not want to save the newContent selector next time

    For the purposes of what the client is looking for, only columnWidth and possibly itemSelector will need to be set.

  3. By default Masonry 0.8 will rearrange elements when the browser is resized. To disable this feature, set the resizable option to false:
    $('#wrap').masonry({columnWidth: 100, resizeable: false })
  4. Gaps will be hard to avoid. There are a couple ways to make sure they don't happen
    1. When item heights are the same height, there's less likely to be any gaps
    2. A greater number of different item widths will produce more gaps. The Space Collective Gallery items can only be two widths.
    3. Do not put outlines or boxes around the items - which emphasize the spaces where there are no items
  5. There are several ways to make the items different sizes:
    1. In the Wordpress template - each different kind of post (quotes, articles, pics, video) would have its own class, and could be styled differently. Quotes would be 2 column, articles 3 column, pics 1 column.
    2. Grid-a-licious sizes items be analyzing any pics in the item. The pic gets sized down to the best fit. Big pics get 3 columns, small pics get 1 column.
    3. You could add into the WordPress template a way for the client to decide how big they want each item. Maybe by selecting adding a tag, or selecting a class from a drop-down menu.

So the big challenges for you will be deciding how you want to size each item. The other challenge is dealing with images.

Masonry works by measuring the size of each item in the wrapping element. If there's an image inside that item and it hasn't fully loaded, then the height of that item will not be calculated. There are two ways to get around this:

  1. Specify the height of each image inline: Grid-a-licious does this via PHP
  2. Call masonry once all images have loaded. Take a look at the Box100 example for more details.

As you'll see in the demos, it's best to call masonry at the bottom of the page, so it does not hold up loading any of the site.

There is no option in the plugin for setting a minimum width of the container div. But you can do this via CSS on your own.

Masonry 0.8 is also compatible with the Infinite Scroll plugin. If you're interested I can include another demo to show you how to use the two scripts together.
